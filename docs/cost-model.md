# 技术成本模型与询价口径

更新日期：2026-07-19

本文给出可复算的成本结构，不把电商散件价格或开发板价格当作工业部署报价。正式采购单价在传感器型号、布点数量、安装条件和服务等级冻结前均标记为 `TBD/RFQ`。为回答评审中的商业可行性问题，本文另给出一套明确标注的内部预算假设；它用于量级判断，不是供应商报价，也不能替代现场勘察。

## 数字证据分级

| 级别 | 用途 | 本文处理方式 |
|---|---|---|
| 公开零售参考 | 检查开发样机部件的量级，不用于工业部署总价 | 保留原币种、页面和访问日期，并注明税运和适配缺口 |
| 内部预算假设 | 在型号冻结前做可复算的预算门槛 | 使用人民币和明确数量，逐项列算式；不得称为报价 |
| 正式 RFQ | 形成采购、施工和服务合同价格 | 型号与现场条件冻结后，取得至少两份同口径书面报价 |

公开零售参考只有一个可核验样例：[Seeed Studio SenseCAP S2100 Data Logger](https://www.seeedstudio.com/SenseCAP-S2100-LoRaWAN-Data-Logger-p-5361.html) 页面在 2026-07-19 显示 `USD 72.50`。该产品是支持 RS485/模拟量/GPIO 的 LoRaWAN 数据记录器，不包含本项目所需水位探头、工业网关、供电、安装、标定、税费、跨境运费、质保或车辆接入，因此只证明“开发采集节点存在公开零售量级”，不作为下方任何人民币预算项的供应商来源。

## 成本口径

```text
单站 CapEx =
Σ(设备数量 × 同规格含税到场单价)
+ 安装布线
+ 防水箱 / 供电 / UPS / 防雷
+ 标定
+ 系统集成与验收
+ 试验与安全验证的一次性分摊

年度 OpEx =
通信资费
+ 云资源
+ 巡检维护
+ 复标定
+ 证书 / 密钥运维
+ 备件摊销
+ 软件与车辆接口许可（如适用）

N 年 TCO = 初始 CapEx + N × 年度 OpEx + N 年内计划更新成本
单车年成本 = N 年 TCO / (覆盖车辆数 × N)
```

所有金额必须统一为同一币种，并注明是否含税、运费、安装和质保。供应商报价若口径不同，不能直接放在同一区间比较。

## 可配置输入

以下参数可直接复制到表格。`数量公式` 是计算关系，不是已冻结的部署数量。

| 类别 | 参数 | 数量公式/输入 | 含税单价 | 当前状态 |
|---|---|---:|---:|---|
| 场端感知 | 主水位传感器 | `站点数 × 每站主传感器数` | `TBD/RFQ` | 原理、量程、精度、接口和安装方式待定 |
| 场端感知 | 辅水位传感器 | `站点数 × 每站辅传感器数` | `TBD/RFQ` | 当前遥测契约要求第二路水位；是否异构冗余待定 |
| 场端感知 | 雨量计 | `新增站点数 × 每站雨量计数` | `TBD/RFQ` | 先确认能否复用场站既有设备 |
| 场端状态 | 路线水触/出口/闸机接点 | `关键点位数` | `TBD/RFQ` | 点位和协议待现场勘察 |
| 边缘计算 | 工业边缘网关 | `站点数 × 主备系数` | `TBD/RFQ` | CPU、协议、4G/5G、温度和防护等级待定 |
| 通信 | SIM、以太网或专网接入 | `链路数` | `TBD/RFQ` | 主备链路和流量模型待定 |
| 供配电 | 电源、UPS、防雷、防水箱 | `站点数或点位数` | `TBD/RFQ` | 取电、续航和安装环境待定 |
| 施工 | 支架、线缆、开孔、土建和安装 | `现场工程量` | `TBD/RFQ` | 必须勘察后报价 |
| 质量 | 首次标定与周期复标定 | `传感器数 × 标定频次` | `TBD/RFQ` | 标定规范与周期待定 |
| 平台 | API、数据库、日志和备份 | `站点数、请求量、留存期` | `TBD/RFQ` | 当前样机可本地运行，生产 SLA 待定 |
| 安全 | 设备证书、密钥托管、审计 | `设备数、证书数、留存期` | `TBD/RFQ` | MVP API Key 不能作为量产认证成本依据 |
| 车辆接入 | 官方 API/SDK、证书和车辆绑定 | `车辆数或调用量` | `TBD/RFQ` | 当前没有官方商务与技术报价 |
| 验证 | HIL、封闭场地、故障注入和实车测试 | `测试轮次和场地天数` | `TBD/RFQ` | 进入车辆控制前的必要预算项 |
| 运维 | 巡检、备件、通信和云服务 | `站点数 × 年` | `TBD/RFQ` | SLA 和备件率待定 |

## 三阶段预算表

| 成本项 | 软件样机 | 试点单站 | 规模部署 |
|---|---:|---:|---:|
| 新增水位/雨量硬件 | `0`（使用场景数据时）或 `TBD/RFQ` | `TBD/RFQ` | `站点数 × 单站 BOM` |
| 边缘网关 | 可复用开发电脑，不计作工业部署成本 | `TBD/RFQ` | `站点数 × 主备配置` |
| 安装、防护和供电 | 不适用 | `TBD/RFQ` | 按现场工程量汇总 |
| 后端与日志 | 本地 FastAPI + SQLite | `TBD/RFQ` | 按请求量、留存期和 SLA 计算 |
| 车辆接口 | 当前 `record-only`，无车辆控制费用依据 | `TBD/RFQ` | 按官方许可与车辆数计算 |
| 测试与安全验证 | 自动化测试已有，实车费用未发生 | `TBD/RFQ` | 按 HIL、场地、车辆和回归轮次计算 |
| 年度运维 | 不作为部署口径 | `TBD/RFQ` | `站点数 × 单站年度 OpEx` |
| 总计 | 不作为商业部署报价 | `Σ 试点项` | `Σ 规模项 + 共性平台/验证成本` |

软件样机中的 `0` 仅表示使用仓库内场景数据时不新增现场硬件，不表示现有电脑、网络或开发工时没有成本。

## 内部预算基线：单站 `record-only` 试点

这组数字只用于评审预算门槛。假设一个 200 车位地下车库、一个出口、四个关键路线接点；部署两路水位传感器、一台雨量计和一台非冗余网关；复用现有市电且现场蜂窝网络可用；遥测周期 5 秒、日志留存 180 天；车辆侧继续为 `record-only`。所有金额均为人民币，按“含税到场预算预留”理解，安装已单列；传感器型号、税率、运费、质保和现场条件尚未由供应商确认，所以对应正式报价仍是 `TBD/RFQ`。

### 一次性 CapEx

| 预算项 | 数量 | 内部预算单价 | 小计 | 正式采购状态 |
|---|---:|---:|---:|---|
| 主/辅水位传感器 | 2 | ¥3,000 | ¥6,000 | `TBD/RFQ`，优先异构冗余 |
| 雨量计 | 1 | ¥1,500 | ¥1,500 | `TBD/RFQ`，先核实现有设备复用 |
| 路线水触/出口状态接点 | 4 | ¥300 | ¥1,200 | `TBD/RFQ`，点位待勘察 |
| 工业边缘网关 | 1 | ¥5,000 | ¥5,000 | `TBD/RFQ`，试点暂不含主备 |
| 蜂窝/以太网接入设备 | 1 | ¥1,200 | ¥1,200 | `TBD/RFQ` |
| 电源、UPS、防雷和防水箱 | 1 套 | ¥3,500 | ¥3,500 | `TBD/RFQ` |
| 支架、线缆和现场安装 | 1 批 | ¥8,000 | ¥8,000 | `TBD/RFQ`，不含新增土建 |
| 首次标定 | 3 台传感器 | ¥800 | ¥2,400 | `TBD/RFQ`，规范待冻结 |
| 本站集成、联调和验收 | 1 批 | ¥10,000 | ¥10,000 | 内部工时预算，合同口径 `TBD/RFQ` |
| **CapEx 合计** |  |  | **¥38,800** | `6,000 + 1,500 + 1,200 + 5,000 + 1,200 + 3,500 + 8,000 + 2,400 + 10,000` |

### 年度 OpEx

| 预算项 | 数量 | 内部预算单价 | 年小计 | 正式采购状态 |
|---|---:|---:|---:|---|
| 通信链路 | 1 年 | ¥1,200 | ¥1,200 | `TBD/RFQ` |
| 云资源、日志和备份 | 12 月 | ¥300 | ¥3,600 | `TBD/RFQ`，按 180 天留存假设 |
| 季度巡检 | 4 次 | ¥800 | ¥3,200 | `TBD/RFQ` |
| 年度复标定 | 3 台 | ¥600 | ¥1,800 | `TBD/RFQ` |
| 证书、密钥和审计运维 | 1 年 | ¥1,000 | ¥1,000 | `TBD/RFQ` |
| 备件年度摊销预留 | 1 批 | ¥2,000 | ¥2,000 | 内部预留，备件清单待冻结 |
| **年度 OpEx 合计** |  |  | **¥12,800** | `1,200 + 3,600 + 3,200 + 1,800 + 1,000 + 2,000` |

因此，这套内部基线的 3 年 TCO 为：

```text
¥38,800 + 3 × ¥12,800 = ¥77,200
按 200 个车位计算：¥77,200 / (200 × 3) = ¥128.67 / 车位年
```

“车位年”只是容量归一化指标，不代表每辆车都会安装硬件，也不是收费建议。

## 内部预算基线：10 站规模样例

规模样例假设每站仍为 200 个车位，不预设批量折扣，并显式增加共享平台和跨站验收预算：

| 预算项 | 算式 | 3 年金额 |
|---|---:|---:|
| 10 站基础 CapEx | `10 × ¥38,800` | ¥388,000 |
| 共享部署、监控和备份初始化 | `1 × ¥50,000` | ¥50,000 |
| 跨站培训与验收 | `1 × ¥30,000` | ¥30,000 |
| 10 站基础 OpEx | `3 × 10 × ¥12,800` | ¥384,000 |
| 中央监控与备份运维 | `3 × ¥24,000` | ¥72,000 |
| **3 年 TCO** | `388,000 + 50,000 + 30,000 + 384,000 + 72,000` | **¥924,000** |

按 2,000 个车位归一化，`¥924,000 / (2,000 × 3) = ¥154.00 / 车位年`。该样例没有使用未经验证的规模折扣；正式 RFQ 后可将每个预算单价替换为同规格含税到场价并直接复算。

## 未计价但不得忽略的项目

下列项目没有可靠资料或书面报价，均保持 `TBD/RFQ`，没有按零元纳入上述 `¥77,200` 或 `¥924,000`：

1. 小鹏官方车辆 API/SDK、系统签名、证书、车辆绑定和调用许可。
2. HIL、封闭场地、测试车辆、功能安全、网络安全、故障注入和实车回归。
3. 新增土建、长距离开槽布线、泵房/闸机改造、地下蜂窝补盲和双路供电。
4. 量产级主备网关、异地容灾、7×24 值守、SLA 赔付和更长日志留存。
5. 样机之前的产品研发、算法研发、合规认证和持续软件支持。

因此当前能得到的是 `record-only` 场端风险决策系统的预算量级，而不是车辆自动迁移系统的完整商业报价。只要上述任一项成为验收范围，就必须先询价并加入 TCO，再判断商业可行性。

## 报价前必须冻结的假设

1. 主/辅水位传感器的测量原理、量程、精度、响应时间、防护等级、输出接口和安装方式。
2. 雨量计与地磁/车位系统是否复用场站现有设备，以及数据接口和使用费用。
3. 网关 CPU、操作系统、采集协议、4G/5G、供电、UPS、防护等级和工作温度。
4. 单站车位数、关键积水点、布点数、线缆距离、取电条件、土建和防雷条件。
5. 云部署区域、站点数、遥测频率、峰值请求量、日志留存、备份策略和 SLA。
6. 设备证书、密钥托管、SIM 流量和专网的计费方式。
7. 小鹏官方车辆 API/SDK、证书、车辆绑定和调用是否收费；未取得书面口径前保持 `TBD/RFQ`。
8. HIL、封闭场地、功能安全、网络安全和实车验证的轮次与验收标准。

## 询价与区间生成规则

1. 型号和数量冻结后，对关键硬件取得至少两份同规格、同地区、同税运口径且仍在有效期内的书面报价。
2. 对安装、布线和土建进行现场勘察，硬件单价与工程费分开列示。
3. 分别计算样机、试点单站和规模部署，不用消费级开发板价格外推工业网关成本。
4. 区间下限只能使用满足全部规格的最低有效报价；上限应包含已确认的主备、备件和工程风险，不用任意百分比填充。
5. 对未取得正式报价的车辆接口、验证和运维项继续标记 `TBD/RFQ`，不把它们按 `0` 计入总价。

## 主要敏感项

| 变量 | 对成本的影响 | 评审时应说明 |
|---|---|---|
| 传感器是否复用 | 直接改变新增 BOM、施工和标定费用 | “复用”必须有协议、精度和数据授权依据 |
| 主辅传感器是否异构 | 影响可靠性、备件和集成复杂度 | 同型号双路不能覆盖全部共因失效 |
| 有线或蜂窝通信 | 影响布线 CapEx 与长期资费 OpEx | 地下车库还需现场验证覆盖和备用链路 |
| 日志留存与 SLA | 影响数据库、备份和运维成本 | 评审版应给出留存天数和可用性目标 |
| 车辆控制接入 | 可能引入许可、证书、测试和责任成本 | 当前公开资料不足，不纳入确定总价 |
| 实车安全验证 | 通常不是单一硬件费用 | 必须单列 HIL、封闭场地、故障注入和回归成本 |

该模型的输出应是一组带假设和来源的可复算区间，而不是一个脱离型号、数量和现场条件的总价。
