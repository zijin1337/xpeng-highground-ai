# 高地 AI（XPENG HighGround）

面向暴雨内涝场景的车辆安全位移概念验证：系统不只回答“哪里可能有水”，还要回答“这辆车什么时候必须离开、路线是否仍然干燥、是否满足移动条件，以及何时必须明确说 No-Go”。

仓库提供一个无需实车、无需后端、无需第三方依赖的交互仿真页面，以及一个可独立测试的确定性决策引擎。它适合直接部署到 GitHub Pages，用来说明从感知、预测、决策到动作授权和事件留痕的完整物理 AI 闭环。

![高地 AI 交互仿真界面](./assets/highground-demo.png)

> 安全声明：本项目仅用于概念验证，不连接、不控制真实车辆，也不代表任何现有或未来量产功能。任何实车应用均需独立安全壳、封闭场地验证、法规与功能安全流程、人工接管和制造商批准。

## 在线演示内容

- 六类可复现场景：日常守望、暴雨快速上涨、传感器冲突、迁移路线受阻、车内有人、移动中通信异常。
- 一键车辆迁移动画：自动进入暴雨演示场景，依次呈现安全自检、单次授权、低速平移、坡道迁移、高位锁车和事件留痕；同步显示实时速度与进度。
- 五个连续输入：降雨强度、主水位、交叉水位、上涨速度、证据置信度。
- “七道闸”与附加安全条件：干燥路线、出口可用、车内无人、充电枪拔除、车辆健康、定位在线、通信在线、远程安全员在线。
- 单次车主授权：授权只对当前事件有效；即使进入迁移窗口，未授权也不会获得移动权限。
- 每分钟推进：水位按上涨速度演进，决策、风险等级和最晚安全启动窗口同步更新。
- 可解释证据链：每一项检查都给出通过/未通过及原因，并生成输入快照哈希和事件编号。
- 失效降级：证据不一致时只提醒；路线见水、乘员未清空等条件触发 No-Go；移动中通信或关键链路异常触发最小风险停车。

## 30 秒运行

项目没有运行时依赖。使用任意本地静态服务器即可：

```bash
python -m http.server 8000
```

浏览器打开 `http://localhost:8000`。

如果使用 Node.js：

```bash
npm run serve
```

## 决策模型

核心时间窗口使用以下可解释公式：

```text
T_last = T_threshold - T_route - T_queue - T_buffer
```

- `T_threshold`：按当前水位与上涨速度估算到达禁行水位的剩余时间。
- `T_route`：以不超过 5 km/h 的封闭场地限速估算干燥路线所需时间。
- `T_queue`：多车分批迁移带来的排队时间。
- `T_buffer`：用于覆盖定位、闸机、路径和通信不确定性的安全余量。

决策顺序固定为：

```mermaid
flowchart LR
    A[输入有效性] --> B[移动中异常]
    B --> C[多源证据一致性]
    C --> D[物理安全闸]
    D --> E[最晚安全启动窗口]
    E --> F[车主单次授权]
    F --> G[动作或保持]
```

安全条件优先于时间窗口：时间再紧，只要路线见水、车内有人、充电未断开、车辆故障或人工兜底不可用，结果仍是 No-Go。

## 决策输出

| 代码 | 页面显示 | 动作权限 |
|---|---|---|
| `STAY` | 原地守望 | 无需移动 |
| `WATCH` | 增强监测 | 无需移动 |
| `PREPARE` | 建议准备迁移 | 等待车主授权 |
| `MIGRATE_NOW` | 建议立即迁移 | 安全闸全通过且单次授权后才授予 |
| `VERIFY_ONLY` | 只提醒，等待复核 | 禁止移动 |
| `NO_GO` | 禁止迁移 | 禁止移动 |
| `EMERGENCY_STOP` | 异常停车并转人工 | 禁止继续移动 |
| `SYSTEM_HOLD` | 输入异常，系统保持 | 禁止移动 |

## 上传到 GitHub

在本目录打开终端，执行：

```bash
git init
git add .
git commit -m "feat: add XPENG HighGround AI concept simulator"
git branch -M main
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main
```

然后进入 GitHub 仓库：

1. 打开 **Settings → Pages**。
2. 将 **Source** 设为 **GitHub Actions**。
3. 等待 `deploy-pages` 工作流完成。
4. Pages 页面会显示公开演示地址。

## 技术边界与后续验证

当前仓库证明的是决策逻辑与交互闭环，不证明车辆已经具备暴雨自动迁移能力。继续进入真实验证前，至少需要：

1. 数字孪生与封闭水槽中的传感器误差、反光、喷溅、污损和断网注入测试。
2. 固定液位计、车辆感知、场地高程和人工巡检之间的交叉标定。
3. 认证封闭停车场内的干燥路线、低流量、远程安全员与一次任务授权。
4. 水触、见水、路线阻塞、乘员存在或任一关键链路失效时的禁行与最小风险停车。
5. 事件黑匣子、版本追溯、权限撤销、保险与责任归属机制。
